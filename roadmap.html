NavTable
isn't finished at all. It's an extension in a constant evolution. There's a roadmap with fixes and new functionalities that will be add in next versions of the program. Nowadays our development plan has three iterations. Feel free to send us your suggestions, bugs, ideas, etc. and we will try to add it in future NavTable releases.

NavTable 0.2.2

On the next revision some of the features and fixes suggested by the community.

  • Assign long attribute names with a text plain file.
  • Better sync with features selections on the gvSIG's view
  • It shows length (perimeter) and area of the geometry
  • Show a warning message when there's unsaved data.
  • Keyboard shortcuts (LEFT/RIGHT/HOME/END) to navigate when focus on the table
  • The 'copy button' will copy values of the selected element instead of the previous one.
  • Table refresh after saving changes in order to check that the changes are correctly saved.
  • Show an error message when there's no features on the active layer.

NavTable 1.0

This version's main feature is the configuration of NavTable's behaviour when it's showing data. The user will be able to set up options such as the fields to be shown, hide empty and null values, etc. XML file will be used to configure some improvement such long attribute names, tooltips, default values, checks, etc.

There will be geometry edition capabilities, and NavTable will work with data tables that don't belong to layers.

And there will be changes from development point of view too. The data access logic will be sepparated from the user graphic interface, so it will be possible to adapt NavTable to another GIS programs.

NavTable 2.0

NavTable will be able to modify the table schema, it will be possible to add, erase and modify attributes, and its order.

Other features will be added too, such as the possibility of automaticaly launch when a new feature has been created, order the record set by a given field or seeking improvements with incremental lexical searches.
